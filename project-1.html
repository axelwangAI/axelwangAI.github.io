信贷 A 卡模型迭代：0.5% 坏账率降幅

采用 XGBoost 模型，通过工程化特征处理，实现资产质量和通过率的精细化平衡。

技术栈：Python / XGBoost / Feature Store 成果：Gini提升12% 周期：6个月

1. 项目背景与挑战

在信贷周期早期，旧模型无法准确捕捉借款人的长期信用风险。坏账率的微小波动对业务盈利能力产生巨大影响，我们面临在不牺牲通过率的前提下，降低风险的挑战。

2. 核心解决方案：XGBoost + 工程化特征

我们没有盲目追求更复杂的模型，而是将 70% 的精力投入到特征工程。核心贡献包括：

  • **时间序列特征:** 提取近 3/6/12 个月负债波动、逾期趋势等高区分度特征。
  • **特征稳定性监控:** 部署自动化监控，确保模型上线后输入特征的 PSI 保持稳定。
  • **模型部署:** 使用 Python Flask 封装模型，实现毫秒级 API 响应。

3. 量化成果

通过新模型，我们在保持原有通过率±5%的范围内，成功将坏账率（M3+）降低了 **0.5个百分点**，模型区分度 Gini 系数从 0.58 提升到 0.70。
